O programa contém todas as funcionalidades propostas no enunciado do trabalho e algumas extras como por exemplo:

- Ter várias lojas e poder aceder a qualquer uma ao iniciar o programa
- Ter uma loja protegida com uma pass criada na criação da loja
- Editar produtos
- Criacao de transacoes com quantidades de produtos
- Bastantes mini-funcoes uteis como por exemplo: verificar se uma data existe mesmo num calendario (vai ao ponto de analisar os "29 de fevereiro")

Grande parte do código é feito em verificações de inputs e até a alteração do próprio input do utilizador para que fique de uma maneira standard, como por exemplo:

- Na criacao de clientes, qualquer letra minúscula no início da frase é convertida para maiúscula quer seja ou nao minúscula
- No input de datas, se o utilizador escrever algo do genero de : 1/1/2016 ou 1/01/2016, é convertido na forma standard 01/01/2016 (DD/MM/YYYY)

Relativamente ao armazenamento de dados utilizamos:

-vetores
-ficheiros de texto
-pares
-ints, chars, strings, etc.
-maps
-multimaps
etc.


A publicidade é feita com recurso a multimaps e vetores, sendo que, no exemplo da publicidade à vítima esta segue esta ordem:

-identificacao dos melhores clientes (clientes que têm o máximo de produtos iguais à vitima)
-identificacao do/dos melhor/es item/ns via comparacao entre os produtos comprados pelos melhores clientes (excluindo os produtos comprados pela vitima)

Nos bottom10, seguimos as instrucoes dadas pelo trabalho.
